SPARKPANEL — УСТАНОВКА НА VDS (КОПИРУЙ КОМАНДЫ ПО ПОРЯДКУ)

ВАЖНО: вместо 203.0.113.10 подставь свой IP ВДС. Все команды выполняй под root или с sudo.

0) НАЙТИ ПАПКУ ПРОЕКТА
cd ~
ls -la
ls -la ~/SparkPanel || true
ls -la ~/SparkPanel/SparkPanel || true
# Должны увидеть папки backend и frontend в одной директории (обычно: ~/SparkPanel/SparkPanel)

1) BACKEND: ПЕРЕЙТИ В ПАПКУ
cd ~/SparkPanel/SparkPanel/backend
pwd
cat package.json | head -n 5
# Должно показать: "name": "sparkpanel-backend"

2) СОЗДАТЬ .env (замени IP ниже на свой)
cat > .env << 'EOF'
PORT=8080
NODE_ENV=production
BASE_URL=http://203.0.113.10:5173
API_BASE_URL=http://203.0.113.10:8080
DATABASE_URL=postgresql://postgres:postgres@localhost:5432/sparkpanel?schema=public
JWT_ACCESS_SECRET=change_me_access_secret_123
JWT_REFRESH_SECRET=change_me_refresh_secret_123
JWT_ACCESS_TTL=15m
JWT_REFRESH_TTL=30d
SMTP_HOST=
SMTP_PORT=587
SMTP_USER=
SMTP_PASS=
SMTP_FROM=SparkPanel <no-reply@sparkpanel.local>
FILES_ROOT=/var/lib/sparkpanel/servers
DISK_QUOTA_ENFORCE=false
DISK_QUOTA_DEFAULT_MB=10240
BACKUP_RETENTION_DAYS=14
BACKUP_RETENTION_MAX=10
S3_ENABLED=false
S3_ENDPOINT=
S3_REGION=us-east-1
S3_BUCKET=
S3_ACCESS_KEY_ID=
S3_SECRET_ACCESS_KEY=
S3_SSE=
S3_PATH_STYLE=false
ADMIN_EMAIL=
ADMIN_PASSWORD=
EOF

3) УСТАНОВИТЬ ЗАВИСИМОСТИ BACKEND (и Prisma)
npm config set registry https://registry.npmjs.org/
npm install --no-fund --no-audit
npx prisma generate
npx prisma migrate deploy

# ЕСЛИ спросит установить другую версию prisma — ОТМЕНИ и убедись, что ты в папке backend, затем повтори npm install и npx prisma generate.

4) ЗАПУСТИТЬ BACKEND (оставь окно открытым)
npm run dev

5) FRONTEND: ОТКРЫТЬ НОВОЕ ОКНО И УСТАНОВИТЬ
cd ~/SparkPanel/SparkPanel/frontend
pwd
cat package.json | head -n 5
# Должно показать: "name": "sparkpanel-frontend"
npm config set registry https://registry.npmjs.org/
npm install --no-fund --no-audit
npm run dev

6) ОТКРЫТЬ В БРАУЗЕРЕ (замени IP)
Панель:  http://203.0.113.10:5173
API:     http://203.0.113.10:8080

7) UFW (если включён файрвол)
sudo ufw allow 5173
sudo ufw allow 8080
sudo ufw allow 25565/tcp
sudo ufw status

8) ПАПКА ДЛЯ ДАННЫХ СЕРВЕРОВ
sudo mkdir -p /var/lib/sparkpanel/servers
sudo chown -R $USER:$USER /var/lib/sparkpanel/servers

9) DOCKER ДОЛЖЕН РАБОТАТЬ
# Если docker не установлен, поставить и добавить пользователя в группу docker
docker ps || (curl -fsSL https://get.docker.com | sudo sh && sudo usermod -aG docker $USER && newgrp docker && docker ps)

10) БЫСТРЫЕ КОМАНДЫ ДЛЯ ПОЛЬЗОВАТЕЛЕЙ (исполнить в папке backend)
# Создать пользователя
cd ~/SparkPanel/SparkPanel/backend
node scripts/user.mjs create --email admin@example.com --username admin --password StrongPass123
# Выдать права ADMIN
node scripts/user.mjs admin --login admin@example.com
# Сброс пароля
node scripts/user.mjs reset-password --login admin@example.com --password NewStrongPass123

ОШИБКИ И РЕШЕНИЯ
- "cd: backend: No such file or directory": не в корне проекта. Иди в ~/SparkPanel/SparkPanel.
- "Cannot find module … / 404 registry": выполни npm config set registry https://registry.npmjs.org/ и проверь, что ты именно в backend или frontend перед npm install.
- "npx prisma generate просит v6": сначала npm install в backend, потом npx prisma generate.
- Не открывается по localhost: мы используем IP. BASE_URL и API_BASE_URL должны быть на IP/домен, НЕ localhost.
- Node нужен 20+: проверка node -v (и npm 10+).

ГОТОВО. После этих шагов панель должна работать по адресу IP:5173.
