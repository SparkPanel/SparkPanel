# Участие в проекте

Спасибо, что вы решили внести свой вклад в SparkPanel! Ниже приведены рекомендации по участию в проекте.

## Перед началом работы

Перед тем как начать работать над новой функцией или исправлением ошибки, пожалуйста:

1. Ознакомьтесь с текущими [открытыми задачами](https://github.com/SparkPanel/SparkPanel/issues)
2. Создайте отдельную ветку для вашей задачи: `git checkout -b feature/telegram-2fa`
3. Регулярно обновляйте вашу ветку: `git fetch origin && git rebase origin/main`

## Работа с кодом

### Двухфакторная аутентификация через Telegram

SparkPanel поддерживает двухфакторную аутентификацию через Telegram. При разработке новых функций, связанных с этой функцией, необходимо:

1. Следовать принципам безопасности при работе с токенами и персональными данными пользователей
2. Использовать middleware для проверки наличия Telegram бот токена и включения двухфакторной аутентификации
3. Реализовывать логирование ошибок при работе с Telegram API
4. Предусматривать обработку исключений при взаимодействии с внешними API

### Тестирование

Для всех изменений, связанных с Telegram 2FA, должны быть написаны:

1. Unit-тесты для валидации данных
2. Integration-тесты для проверки работы с Telegram API
3. Тесты для middleware и контроллеров

Тесты должны покрывать как успешные сценарии, так и различные варианты ошибок:
- Отсутствие Telegram бот токена
- Неверный формат chat_id
- Ошибки при отправке сообщения через Telegram API

## Пул-реквесты

При создании pull request:

1. Убедитесь, что все тесты проходят успешно
2. Добавьте описание изменений на русском и английском языках
3. Укажите, какие части проекта были изменены
4. Если это касается безопасности, отметьте это в заголовке PR

## Архитектурные требования

При разработке новых функций, связанных с Telegram 2FA, следует соблюдать следующие архитектурные принципы:

1. Логика работы с Telegram API должна находиться в отдельных классах сервисов
2. Все чувствительные данные (токены, chat_id) должны обрабатываться безопасным образом
3. Не должно быть дублирования логики в разных частях проекта
4. Код должен соответствовать стандартам PSR-12 и TypeScript ESLint

## Безопасность

При работе с Telegram 2FA особенно важно соблюдать принципы безопасности:

1. Никогда не логируйте токены или chat_id в чистом виде
2. Валидируйте и очищайте все входящие данные
3. Используйте HTTPS для всех запросов к Telegram API
4. Обеспечьте защиту от спама и злоупотребления системой двухфакторной аутентификации
